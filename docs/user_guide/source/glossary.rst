Glossary
========

.. _glossary:

.. glossary::
    :sorted:

    API (Application Programming Interface)
        The software layer exposed to external systems.
        In the context of Scrutiny, it refers to the server-side interface that receives and processes client requests.

    CAN (Controller Area Network)
        A multi-node communication protocol widely used in the automotive industry.

    CAN-FD  (CAN Flexible Datarate)
        An enhanced version of the CAN protocol that supports higher data throughput.

    CLI  (Command Line Interface)
        Software designed to be executed from a terminal.

    CSV  (Comma Separated Values)
        A text-based data format representing tabular data, where columns are separated by commas and rows by line breaks.

    DAP  (Debug Adapter Protocol)
        A communication protocol used for microcontroller debugging.

    ELF  (Executable and Linkable File)
        A file format containing compiled code. In this context, it mostly refer to the embedded firmware.

    GUI  (Graphical User Interface)
        A window-based software interface intended for use with a mouse or other pointing device.

    HIL  (Hardware In the Loop)
        A testing methodology that involves interaction with a physical device.

    I²C  (Inter-Integrated Circuit)
        A multi-node, master/slave communication protocol requiring two wires. Commonly used within a single electronic board.

    JLink
        A debug probe manufactured by SEGGER Microcontroller.

    JSON  (JavaScript Object Notation)
        A text-based data format capable of representing complex structured objects.

    JTAG  (Joint Test Action Group)
        A debugging communication protocol. The term is sometimes used as an umbrella designation for various debug interfaces.

    QT
        A widely used graphical user interface framework developed by The Qt Company and used by the Scrutiny GUI.

    RPV  (Runtime Published Value)
        A type of server-exposed element that is not a variable but is explicitly defined within the firmware.

    RTT  (Real Time Transfer)
        A communication protocol developed by SEGGER Microcontroller that operates on top of lower-level debug interfaces such as SWD, JTAG, ICSP, and others.

    SDK  (Software Development Kit)
        A software library used to interact with the server.

    SFD  (Scrutiny Firmware Description)
        A custom file format containing metadata and structural information for a specific firmware revision.

    SPI  (Serial Peripheral Interface)
        A multi-node, master/slave communication protocol that requires a dedicated signal per node. Typically used within a single electronic board.

    SWD  (Serial Wire Debug)
        A debugging communication protocol developed by Arm and used on Arm cores.

    TCP  (Transmission Control Protocol)
        A network protocol providing a reliable byte stream, used for communication between clients and the server.

    UDP  (User Datagram Protocol)
        A network protocol providing unreliable datagram-based communication.
